💡 The Spark

A few nights ago, I opened Typora and was reminded of why I loved it — the clean interface, the beautiful typography, the distraction-free writing. But I had 15 days left on my trial, and instead of reaching for my wallet, I asked myself: "Could I build something better?"

Not just a clone — something that addressed Typora's limitations:

  • Only one editing mode (WYSIWYG-only can feel limiting)
  • No split-pane view for learning markdown syntax
  • Closed source and $14.99 price tag
  • Heavy memory footprint (1011 MB on my system)
  • Poor Linux integration (no tiling window manager support)

So I fired up my IDE and started what became Docura — a Markdown editor built in 37 hours that uses 23% less memory and adds features Typora can't match.

⚙️ Three Modes, One Editor

📝
Code Mode

Pure markdown editing with Monaco Editor. Full VS Code experience with syntax highlighting for 12 themes.

✍️
Live Mode

Split-pane WYSIWYG with toolbar buttons. See markdown syntax and rendered output simultaneously.

👁️
Preview Mode

Beautiful read-only rendering with Typora-inspired typography and interactive checkboxes.

🏗️ The Tech Stack

Built with modern, performant technologies:

Tauri 2.8 (Rust) React 19 Vite 7 Monaco Editor WebKitGTK react-markdown remark-gfm jspdf

Why Tauri + React?

  • WebKitGTK vs Chromium: Tauri uses system WebKit instead of bundling Chromium. Result? 780 MB vs 1011 MB — 231 MB saved.
  • Rust Backend: Native performance and security. File I/O and system integration in Rust, not Node.js.
  • Native Integration: Respects system themes, uses native dialogs, integrates with tiling window managers.
